# Dry Bean 多分类机器学习期末项目设计规格

## 1. 项目目标

基于教师提供的 Dry Bean 脏数据训练集、验证集和测试集，完成一个围绕课程评分标准设计的全流程机器学习项目。项目覆盖数据分析、数据清洗与特征工程、多算法实验、命令行运行、Streamlit 系统展示、GitHub README 和可编辑 Word 论文。

本项目采用“评分定向版”范围：完整覆盖评分项和高价值加分项，不建设在线 API、数据库、Docker、登录系统或复杂前端。

## 2. 已知数据概况

- 训练集：9,527 行，17 列。
- 验证集：1,347 行，17 列。
- 测试集：2,737 行，17 列。
- 输入包含 16 个干豆形态数值特征，目标为 7 个真实类别。
- `Perimeter` 和 `Solidity` 存在缺失值。
- 训练集存在 21 条完全重复记录。
- 标签存在大小写不一致、尾随空格和数字替代字母等污染，例如 `dermason`、`DERMASON `、`D3RMAS0N`。
- 类别分布不完全均衡，因此除 Accuracy 外使用 Macro-F1 补充评价。

## 3. 数据处理设计

### 3.1 防止数据泄漏

训练集用于拟合所有数据处理参数和模型；验证集用于模型选择与超参数比较；测试集只用于最终评价。缺失值填补、标准化和其他依赖数据分布的变换只在训练集上拟合，再应用到验证集和测试集。

### 3.2 标签清洗

标签清洗按以下顺序执行：

1. 去除首尾空白。
2. 统一转换为大写。
3. 修复明确的数字替字污染，如 `0 → O`、`3 → E`。
4. 将清洗结果限制在 7 个合法类别集合中；无法映射的标签触发错误并输出样本位置。

### 3.3 特征清洗

- 删除训练集完全重复记录，验证集和测试集保留原始行数以维持教师划分。
- 使用训练集各列中位数填补数值缺失。
- 对需要尺度一致的模型使用标准化；树模型使用未标准化数值。
- 检查非有限值、明显非法值和极端离群值。
- 对极端值采用基于训练集分位数的截尾处理，阈值固定后应用于其他数据集。

### 3.4 特征工程

在原始特征基础上构建少量具有几何意义的衍生特征，例如：

- 凸包空隙：`ConvexArea - Area`
- 轴长差：`MajorAxisLength - MinorAxisLength`
- 面积与凸包面积之比
- 周长与等效直径之比
- 若干形状因子的稳定组合

衍生特征必须避免与已有特征完全重复。通过消融实验比较：

- 清洗后的原始特征
- 清洗后的原始特征加衍生特征

只有实验结果支持时，才在最终模型中保留衍生特征。

## 4. 算法设计

项目比较以下 5 种多分类算法：

1. 逻辑回归：课堂算法，作为线性基准，并记录迭代损失。
2. K 近邻：课堂算法，作为惰性学习基准，重点分析推理速度。
3. 随机森林：课堂外算法，代表 Bagging 树集成方法。
4. XGBoost：课堂外算法，代表 Boosting 方法，并记录训练集和验证集损失。
5. 手写高斯朴素贝叶斯：自主实现类别先验、各类别特征均值与方差、对数后验和预测过程，不调用现成分类器。

模型选择只使用训练集和验证集。所有模型使用统一随机种子，并保存最终模型和预处理器。

## 5. 实验设计

### 5.1 评分要求内的实验

- 测试集精度：比较 Accuracy，并给出完整结果表。
- Loss 曲线：逻辑回归和 XGBoost 绘制训练过程曲线；不具备迭代训练过程的模型明确标注“不适用”。
- 推理速度：预热后重复批量预测，报告中位耗时和单样本平均耗时。
- 鲁棒性：只污染训练数据，验证集和测试集保持干净；比较模型在不同噪声下的精度下降。
- 过拟合：比较训练集与测试集的 Accuracy 和 Macro-F1 差值。

鲁棒性实验包含四类噪声：

- 高斯特征噪声
- 脉冲/异常点噪声
- 随机缺失噪声
- 标签翻转噪声

每类噪声设置多个由弱到强的等级，并使用相同随机种子保证算法间公平比较。

### 5.2 额外高价值对比维度

只增加以下 4 项，避免无意义堆砌：

- Macro-F1：补充类别不均衡情况下的总体表现。
- 训练时间：与推理速度共同反映计算效率。
- 模型文件大小：反映存储与部署成本。
- 特征工程消融：证明数据处理是否带来真实收益。

### 5.3 结果解释

除总指标外，生成混淆矩阵和类别级指标，用于解释主要误分类模式。随机森林或 XGBoost 的特征重要性用于辅助说明哪些形态特征具有较强判别能力，但不作为新的独立比较维度。

## 6. 软件结构

项目按职责拆分为小型模块：

- 配置模块：路径、随机种子、模型参数和噪声等级。
- 数据模块：加载、数据审计和数据集一致性检查。
- 清洗模块：标签规范化、重复处理、缺失值填补和异常值处理。
- 特征模块：衍生特征与特征消融配置。
- 模型模块：统一模型接口和手写高斯朴素贝叶斯。
- 训练模块：训练、验证、模型保存和 Loss 记录。
- 评估模块：指标、计时、模型大小、混淆矩阵和过拟合分析。
- 鲁棒性模块：噪声注入与多强度实验。
- 可视化模块：生成论文和网页共用的高清图表。
- 报告模块：汇总 CSV、JSON 和论文素材。
- Streamlit 应用：只读取离线实验结果，不在 UI 中重新训练。
- 统一 CLI：支持完整流水线和分阶段执行。

## 7. 命令行行为

至少提供以下命令：

- 完整执行：数据分析、清洗、训练、评估、鲁棒性实验和图表生成。
- 数据分析与清洗。
- 模型训练。
- 最终评估。
- 鲁棒性实验。
- 启动 Streamlit 展示页。

算法运行阶段仅输出命令行日志，不弹出 UI。命令失败时给出清楚的文件、阶段和原因说明。

## 8. Streamlit 与 GitHub 展示

Streamlit 页面包含：

1. 项目概览与数据描述
2. 数据污染与清洗结果
3. 特征分布和相关性
4. 五种算法说明
5. Accuracy、Macro-F1、训练时间、推理速度和模型大小结果表
6. Loss 曲线
7. 鲁棒性对比
8. 过拟合分析
9. 特征工程消融
10. 混淆矩阵、特征重要性和最终结论

README 作为 GitHub 首页，包含项目简介、目录结构、环境安装、统一命令、数据处理说明、算法说明、核心结果表、关键图表、展示截图和论文说明。仓库发布和最终网页链接在本地成果验证完成后处理。

## 9. Word 论文

生成可编辑 `.docx` 论文，采用正式课程报告风格，包含：

- 封面
- 中文摘要与关键词
- 自动目录
- 绪论
- 数据集与污染分析
- 数据清洗与特征工程
- 算法原理与实现
- 实验设计
- 实验结果与多维分析
- 鲁棒性与过拟合分析
- 系统设计和网页展示截图
- 课程总结
- 参考文献

论文图表直接引用实验程序生成的高清图片，结果数字从结构化实验结果自动读取，避免论文与代码结果不一致。最终 DOCX 必须渲染为页面图片并逐页检查中文字体、分页、目录、表格和图像布局。

## 10. 测试与验收

自动测试至少覆盖：

- 标签污染映射。
- 缺失值处理仅在训练集拟合。
- 特征列顺序一致。
- 手写高斯朴素贝叶斯的拟合、概率计算和预测。
- 噪声强度为零时数据不变。
- 各噪声注入函数不修改原始输入。
- 指标和计时输出结构稳定。
- CLI 能运行指定阶段。

最终验收条件：

- 五种算法均能在统一流水线中运行。
- 所有评分要求实验均产生表格或图表。
- 四项额外对比维度均有结果和解释。
- 测试集不参与预处理拟合、调参或模型选择。
- Streamlit 能从已生成结果正常展示。
- README 命令可复现主要成果。
- Word 论文内容、实验数字和系统截图一致。

## 11. 明确不做的内容

- 在线模型服务或 REST API
- 数据库
- 用户登录和权限系统
- Docker 和云端部署流水线
- 复杂前后端分离架构
- 大规模自动超参数搜索
- 概率校准、综合雷达评分等低优先级扩展
